# Traits System Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a composable Traits system with two traits — `Flying` (straight-line movement ignoring streets) and `Vampire` (marker that gates Blood simulation) — and automatically assign `VampireTrait` to all player characters.

**Architecture:** Each trait is a plain object `{ id, update?(char, dt, ctx) }`. `updateCharacter` iterates `char.traits` before its default street-walking logic; if any trait's `update` returns `true`, the tick is claimed and street movement is skipped. Blood simulation in `simulation/blood.js` is switched from filtering on `isPlayer` to filtering on the Vampire trait.

**Tech Stack:** Vanilla JS ES modules, Canvas API, no build step. Verification via `node --check` (syntax) and `node --input-type=module -e` (logic assertions), plus browser smoke with Playwright.

## Task 2: Create `FlyingTrait`

**Files:**
- Create: `entities/traits/flying.js`

- [x] **Step 1: Create the file**

```js
// entities/traits/flying.js

export const FlyingTrait = {
  id: 'flying',

  update(char, dt, { mapWidth, mapHeight }) {
    if (!char.flyTarget) {
      char.flyTarget = {
        x: char.rng.float(0, mapWidth),
        y: char.rng.float(0, mapHeight),
      };
    }

    const dx = char.flyTarget.x - char.pos.x;
    const dy = char.flyTarget.y - char.pos.y;
    const dist = Math.sqrt(dx * dx + dy * dy);

    if (dist < 1) {
      char.flyTarget = null;
      return true;
    }

    const step = char.speed * dt;

    if (step >= dist) {
      char.pos.x = char.flyTarget.x;
      char.pos.y = char.flyTarget.y;
      char.flyTarget = null;
    } else {
      char.pos.x += (dx / dist) * step;
      char.pos.y += (dy / dist) * step;
    }

    return true;
  },
};
```
